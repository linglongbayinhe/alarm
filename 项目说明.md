# 项目说明

## 项目目标
- 基于 `ESP32-WROOM-32E` 实现智能闹钟底座，手机通过 BLUFI 蓝牙下发 Wi-Fi 名称和密码。
- ESP32 连接 Wi-Fi 后启动 SNTP 校时；无网时优先使用 `DS3231` 维持本地时间。
- 通过 `ST7789` SPI 彩屏显示联网状态、日期和时间。

## 核心结构
- 主入口在 `main/blufi_example_main.c`，负责初始化 NVS、BLUFI/Wi-Fi、时间服务、RTC 服务和 UI 刷新任务。
- `main/time_service.c` 负责系统时间、SNTP 启动、本地时区设置和同步完成通知。
- `main/rtc_service.c` 负责 DS3231 的 I2C 初始化、读写和时间有效性判断。
- `main/display_service.c` 负责三行显示视图模型的渲染接口、内置字模、RGB565图片显示和 `esp_lcd + ST7789` SPI 后端。
- `main/display_config.h` 保存当前项目自己的屏幕型号、SPI 引脚、背光引脚、横屏方向、镜像和偏移配置。
- `main/CMakeLists.txt` 需要显式依赖 `esp_driver_i2c`、`esp_driver_gpio`、`esp_driver_spi`、`esp_lcd`、`spiffs` 和 `lwip`。
- `main/DISPLAY_IMAGE_GUIDE.md` 为图片显示功能的详细使用指南。
- `main/alarm2.4.ino` 为旧 Arduino 实验代码，不参与当前 ESP-IDF 构建。

## 当前显示行为
- 当前逻辑显示为横屏，逻辑分辨率为 `320x240`；物理面板仍为 `240x320` 的 ST7789，通过旋转参数映射到横屏。
- 第一行显示右上角 Wi-Fi 图标，不再显示 `WIFI/LOCAL` 文字：
  - 已连上 Wi-Fi 时显示白色 `90°` 扇形 Wi-Fi 图标，并按信号强度显示 `1~3` 级扇形带
  - 未连上 Wi-Fi 时显示同样的白色 `90°` 扇形底图，并叠加红色斜杠
- 第一行图标区域已上移到屏幕顶部，Wi-Fi 图标默认贴近右上边缘显示。
- 第二行显示日期 `YYYY-MM-DD`。
- 第三行显示时间 `HH:MM:SS`。
- 系统时间无效时，第二、三行显示占位值 `---- -- --` 和 `--:--:--`。
- 若屏幕初始化失败，`display_service` 会回退到日志输出，串口里仍能看到 `Screen line1/2/3`。

## 图片显示功能（新增）
- 新增 `display_service_show_image()` 接口，支持从 SPIFFS 文件系统读取 RGB565 格式的全屏图片。
- 新增 `display_service_show_image_data()` 接口，支持从内存（RAM/ROM）显示预加载的图片数据。
- 图片要求：RGB565格式、320×240分辨率、每张约150KB。
- SPIFFS 分配了 ~900 KB 分区，可容纳约6张图片。
- 使用场景示例：启动画面、Logo、欢迎界面等。
- 具体用法见 `main/DISPLAY_IMAGE_GUIDE.md`。

## 关键实现说明
- Wi-Fi 获取 IP 后会启动 SNTP，使用 `pool.ntp.org` 同步系统时间。
- 启动时若 DS3231 可用且时间有效，会先从 RTC 恢复系统时间；SNTP 同步成功后会回写 DS3231。
- 显示层使用单块 DMA 行缓冲。由于 `esp_lcd_panel_draw_bitmap()` 的颜色传输是异步的，缓冲在传输完成前不能被下一次绘制复用。
- 为解决“TFT 文字有时只显示下半部分”问题，`display_service` 通过 `on_color_trans_done` 回调和二值信号量等待每次 SPI DMA 刷新完成后再复用缓冲。
- 横屏模式下若出现整体偏移，优先调整 `main/display_config.h` 中的 `DISPLAY_GAP_X` / `DISPLAY_GAP_Y`，不要改业务渲染坐标。
- 当前 Wi-Fi 状态图标由代码直接绘制为 `90°` 扇形，不再依赖固定 1-bit 位图资源。
- 当前扇形朝向为正上方；位置、半径和粗细参数已在 `main/display_service.c` 宏定义处补充用途注释。
- 当前 `RSSI` 会被映射为 `1~3` 级 Wi-Fi 信号强度，与 3 条扇形带保持一致。
- 第一行图标区域的起始 Y 坐标已改为 `0`，用于把图标推到屏幕顶边附近。
- 为避免扇形顶部出现孤立白点，绘制逻辑会跳过每条扇形带最顶端的单像素尖点。

## 默认硬件参数
- 屏幕：`ST7789`，颜色顺序 `BGR`。
- 屏幕引脚：`BL=26`、`MOSI=16`、`SCLK=4`、`CS=18`、`DC=5`、`RST=17`。
- RTC：`DS3231`，默认使用 `I2C_NUM_0`，`SDA=21`、`SCL=22`。
- 时区固定为 `HKT-8`。
- 若后续项目接线不同，只修改当前项目的 `main/display_config.h`，不改全局库配置。

## 2026-04-09 Wi-Fi Icon Update
- Wi-Fi status icon upgraded again to a `40x28` built-in bitmap so the shape is closer to the provided `wifi.png` style.
- The connected icon now uses three thicker rounded Wi-Fi arcs plus a round dot on the TFT black background.
- The disconnected icon now uses the same white Wi-Fi base plus a thicker red diagonal slash matching the provided `no-wifi.png` style.
- Connected state now maps `RSSI` to `1~3` signal levels; RSSI read failures fall back to the full Wi-Fi icon.
- The top line remains icon-only, while the date and time layout is unchanged.

## 2026-04-10 Wi-Fi Icon Refinement
- The built-in Wi-Fi icon width is narrowed from `40` to `36`, which is the midpoint between the current and previous versions.
- The white signal shape is no longer a `180°` upper semicircle; it has been tightened into a `90°` fan-like sector.
- The `90°` fan is now centered on the upward direction instead of opening toward the upper-right corner.
- The Wi-Fi icon now uses `3` signal bands, matching common Android-style Wi-Fi visuals.
- The three signal bands are narrower than the previous version to reduce the oversized look on the `320x240` TFT.
- The disconnected red slash has been reshaped so the lower-left tail no longer leaves the extra block at its lower-right edge.
- The icon is now rendered procedurally instead of using fixed monochrome bitmaps, so later tuning of angle, radius and slash placement only needs code changes.
